# Руководство по интерфейсу ПР2 и ПР3

## Где найти кнопки

После входа под администратором в верхней панели появляются две новые кнопки:

```
[⚡ Создать автомобиль]  [+ Добавить деталь]  ...  [☰ Справочники]  [⚙ Параметры]  [👤 Новый пользователь]
```

- **Справочники** → управление перечислениями (ПР2)
- **Параметры** → управление параметрами изделий (ПР3)

---

## ПР2 — Справочники перечислений

### Что это

Перечисление — это справочник допустимых значений для какого-либо атрибута.  
Например: «Стандарт зарядки» содержит значения `CCS2`, `CHAdeMO`, `Type 2`.

### Шаг 1 — Создать класс перечисления

1. Нажать **Справочники**
2. Нажать **+ Новый справочник**
3. Заполнить форму:
   - **Название** — например, `Стандарт зарядки`
   - **Тип компонента** — к чему относится: `charger`, `battery`, `emobile` и т.д. (необязательно)
4. Нажать **Сохранить**

### Шаг 2 — Добавить значения (позиции)

1. В списке справочников нажать иконку **👁** рядом с нужным
2. Нажать **+ Добавить значение**
3. Заполнить:
   - **Значение** — например, `CCS2`
   - **Порядок** — число, определяющее место в списке (0, 1, 2…)
4. Нажать **Сохранить**

Повторить для каждого значения.

### Дополнительные возможности

| Кнопка | Что делает |
|--------|------------|
| **По порядку (SQL)** | Вызывает SQL-функцию `get_enum_values` — показывает значения, отсортированные по `order_num` |
| **Проверить значение** | Вызывает `validate_enum_value` — проверяет, есть ли введённое значение в справочнике |
| Иконка ✏️ рядом с позицией | Редактировать значение или изменить его порядок |
| Иконка 🗑️ | Удалить значение |

### Пример для электромобилей

```
Стандарт зарядки (component_type: charger)
├── 0: CCS2
├── 1: CHAdeMO  
└── 2: Type 2

Тип кузова (component_type: body)
├── 0: Седан
├── 1: Хэтчбек
└── 2: Кроссовер

Привод (component_type: emobile)
├── 0: Передний
├── 1: Задний
└── 2: Полный
```

## ПР3 — Параметры изделий

### Что это

Параметры — это характеристики компонентов и автомобилей с числовыми ограничениями.  
Например: «Запас хода» для `emobile`, ограничен от 50 до 800 км.

Окно содержит **три вкладки**:

```
[ Описания параметров ]  [ Параметры компонентов ]  [ Значения автомобилей ]
```

---

### Вкладка 1 — Описания параметров

Здесь создаются шаблоны параметров (что это за характеристика и какого типа).

#### Создать параметр

1. Нажать **Параметры** → открыть вкладку **Описания параметров**
2. Нажать **+ Новый параметр**
3. Заполнить:

| Поле | Описание | Пример |
|------|----------|--------|
| **Обозначение (код)** | Уникальный технический код | `range_km` |
| **Полное название** | Человекочитаемое название | `Запас хода` |
| **Тип** | `real` / `int` / `str` / `enum` | `real` |
| **Единица измерения** | Для числовых | `км` |
| **ID перечисления** | Только для типа `enum` — ID из ПР2 | `3` |

**Типы параметров:**
- `real` — вещественное число: жирность 45.5%, напряжение 400.0 В
- `int` — целое число: масса 1850 кг, количество ячеек 96
- `str` — строка: страна сборки «Германия»
- `enum` — значение из справочника ПР2: стандарт зарядки «CCS2»

#### Примеры параметров

```
range_km     | Запас хода           | real | км
weight_kg    | Снаряжённая масса    | int  | кг
max_speed    | Максимальная скорость| int  | км/ч
voltage      | Напряжение батареи   | real | В
origin       | Страна сборки        | str  | —
charge_std   | Стандарт зарядки     | enum | — (enum_class_id: 1)
drive_type   | Тип привода          | enum | — (enum_class_id: 2)
```

---

### Вкладка 2 — Параметры компонентов

Привязка параметра к типу компонента с ограничениями для числовых значений.

#### Привязать параметр

1. Нажать **+ Привязать параметр**
2. Заполнить:

| Поле | Описание | Пример |
|------|----------|--------|
| **Тип компонента** | Выбрать из списка | `emobile` |
| **ID параметра** | ID из вкладки 1 | `1` |
| **Порядок** | Порядок вывода | `0` |
| **Минимум** | Нижняя граница (для real/int) | `50` |
| **Максимум** | Верхняя граница (для real/int) | `800` |

**Доступные типы компонентов:**
`emobile`, `battery`, `engine`, `chassis`, `charger`, `electronics`, `body`, `frame`, `suspension`

#### Фильтр по типу (SQL-функция)

В поле **«Фильтр по типу»** введи, например, `emobile` и нажми **Фильтр (SQL)**.  
Это вызовет SQL-функцию `get_component_parameters('emobile')` — вернёт полные данные о параметрах именно этого типа.

#### Скопировать параметры между типами

1. Нажать **Скопировать тип**
2. Заполнить:
   - **Источник** — откуда копировать, например `emobile`
   - **Получатель** — куда копировать, например `emobile_sport`
3. Нажать **Скопировать**

Вызывает SQL-процедуру `copy_component_parameters` — копирует все привязки со всеми ограничениями.

#### Пример привязок для `emobile`

```
Тип: emobile
├── range_km  (Запас хода)      real  50–800 км   порядок: 0
├── weight_kg (Масса)           int   500–3000 кг  порядок: 1
├── max_speed (Макс. скорость)  int   60–300 км/ч  порядок: 2
├── charge_std (Стандарт заряд) enum  —            порядок: 3
└── drive_type (Тип привода)    enum  —            порядок: 4
```

---

### Вкладка 3 — Значения автомобилей

Здесь вводятся конкретные значения параметров для каждого автомобиля.

#### Добавить значение

1. Нажать **+ Новое значение**
2. Заполнить:

| Поле | Описание |
|------|----------|
| **ID автомобиля** | ID из каталога (emobile_id) |
| **ID привязки** | ID из вкладки 2 (component_parameter_id) |
| **val_real** | Для параметров типа `real` |
| **val_int** | Для параметров типа `int` |
| **val_str** | Для параметров типа `str` |
| **enum_val_id** | ID позиции перечисления (из ПР2) |

Заполнять нужно только одно поле значения — то, которое соответствует типу параметра.

#### Фильтр по автомобилю

Введи **ID автомобиля** в поле и нажми **По автомобилю** — загрузятся только его параметры.  
Это вызывает эндпоинт `GET /api/emobile-parameter/byEmobile{id}`.

#### Пример значений для автомобиля #1

```
Автомобиль #1 (Tesla Model S аналог)
├── cp#1 (Запас хода)       val_real = 580.0
├── cp#2 (Масса)            val_int  = 2162
├── cp#3 (Макс. скорость)   val_int  = 250
├── cp#4 (Стандарт зарядки) enum_val = 1  (→ «CCS2»)
└── cp#5 (Тип привода)      enum_val = 3  (→ «Полный»)
```

---

## Типичный рабочий процесс

```
1. ПР2: Создать справочники (Справочники → + Новый справочник)
   └── Добавить значения в каждый справочник

2. ПР3 вкладка 1: Создать описания параметров (+ Новый параметр)
   └── Для параметров типа enum — указать ID справочника из шага 1

3. ПР3 вкладка 2: Привязать параметры к типам компонентов
   └── Указать min/max для числовых параметров

4. ПР3 вкладка 3: Ввести значения для конкретных автомобилей
   └── Использовать фильтр по ID автомобиля для просмотра
```

---

## Примечания

- Удаление **класса перечисления** каскадно удаляет все его позиции
- Удаление **описания параметра** удаляет все его привязки к типам и все значения
- Для параметра типа `enum` значение `enum_val_id` — это ID конкретной **позиции** (не класса)
- ID автомобиля можно посмотреть в каталоге — колонка **ID** (первые 8 символов UUID)
